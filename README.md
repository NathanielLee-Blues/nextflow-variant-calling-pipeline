# Nextflow Bacterial Variant-Calling Pipeline

## Project overview

This pipeline converts the manual bacterial variant-calling workflow into a Nextflow analysis with samplesheet-driven inputs and explicit process dependencies.

It uses a real paired-end *Escherichia coli* dataset and the REL606 reference genome, then runs FastQC, BWA, SAMtools, BCFtools and a Python VCF summariser. Nextflow manages the order of execution, intermediate hand-offs and pipeline reports.

## Main result

The filtered VCF contained 407 candidate variants:

| Variant type | Count |
|---|---:|
| SNP | 353 |
| Indel or complex | 54 |
| **Total** | **407** |

The result matches the manual workflow because both implementations use the same data and analytical logic. This agreement checks that the workflow was transferred into Nextflow without materially changing the final summary. It does not validate the biological accuracy of the candidate calls.

## Workflow

```text
samplesheet.csv
        ↓
FastQC
        ↓
BWA alignment
        ↓
SAMtools sorting, indexing and alignment summary
        ↓
BCFtools calling and filtering
        ↓
Python VCF summarisation
        ↓
Results tables and Nextflow reports
```

## Running the pipeline on Ubuntu

Install the required software:

```bash
sudo apt update
sudo apt install -y openjdk-17-jre bwa samtools bcftools fastqc python3
```

Install Nextflow:

```bash
curl -fsSL https://get.nextflow.io | bash
mkdir -p ~/.local/bin
mv nextflow ~/.local/bin/
echo 'export PATH="$HOME/.local/bin:$PATH"' >> ~/.bashrc
source ~/.bashrc
```

Prepare the data and run the pipeline:

```bash
bash scripts/download_data.sh
nextflow run main.nf
```

## Inputs

| File | Description |
|---|---|
| `data/reference/ecoli_rel606.fasta` | REL606 reference genome |
| `data/reads/sample_R1.fastq` | Read 1 FASTQ |
| `data/reads/sample_R2.fastq` | Read 2 FASTQ |
| `samplesheet.csv` | Sample identifiers and input paths |

## Outputs

| Output | Description |
|---|---|
| `results/qc/` | FastQC reports |
| `results/alignment/` | Alignment and SAMtools summaries |
| `results/variants/` | Raw and filtered VCF files |
| `results/tables/called_variants_summary.csv` | Variant-level summary |
| `results/tables/variant_type_counts.csv` | Counts by variant type |
| `results/pipeline_info/` | Timeline, report, trace and workflow DAG generated locally |

## Interpretation

Nextflow improves orchestration and traceability, but it does not make the underlying calls more accurate by itself. Call quality still depends on the reference, input reads, alignment, caller and filters. The 407 records should therefore be interpreted as candidate variants generated by the specified workflow.
